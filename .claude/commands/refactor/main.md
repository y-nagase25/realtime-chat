# Refactor Code

You are tasked with refactoring the specified code or file.

## Refactoring Guidelines

1. **Analysis First**: Analyze the current code structure and identify issues
2. **Explain Changes**: Clearly explain what you're refactoring and why
3. **Maintain Functionality**: Ensure behavior remains exactly the same
4. **Follow Project Standards**: Adhere to the coding standards in CLAUDE.md
5. **Type Safety**: Maintain or improve TypeScript type safety
6. **Test Coverage**: Do not break existing tests

## Target

$ARGUMENTS

## Process

1. Show the current code structure
2. Identify specific issues (complexity, duplication, naming, etc.)
3. Propose refactoring plan
4. Execute refactoring step by step
5. Verify with type checking and linting

## Output

- Before/after comparison
- List of changes made
- Any breaking changes or migration notes
